// capabilities

Everything you need to visualize data in 3D

A focused, zero-boilerplate rendering engine with a builder API. Drop in a closure, get an animated surface.

Cross-Platform GPU Rendering
Built on wgpu — runs on Vulkan, Metal, and DirectX 12 with zero platform-specific code on your end.
Animated Surfaces
Time-varying meshes driven by f(x, z, t) → y closures. Every frame resamples your function automatically.
Static Wireframe Plots
Render any f(x, z) → y as a wireframe mesh with automatic Y-gradient coloring. Multiple graphs per scene.
Scatter Plots
Efficient point-cloud rendering via a dedicated point-list pipeline, separate from surface meshes for correct compositing.
Orbit Camera
Spherical-coordinate camera with drag-to-orbit and scroll-to-zoom. Pitch clamped to avoid gimbal lock. Aspect-correct projection.
Legend Overlay
Per-graph labels rendered with GPU-accelerated glyphon text. Configure labels and colors via PlotConfig.

Gaussian ripple wave

The animation running in main.rs — a Gaussian-enveloped cosine wave propagating outward over time. The canvas on the left is a live JavaScript reproduction of the same math.

main.rs 
let plot_data = PlotData::new()
    .with_config(config)
    .add_animated_graph(
        range.clone(), range.clone(),
        |x, z, t| {
            let r = (x*x + z*z).sqrt();
            let width = 2.0;
            let gaussian = (-(r*r) / (2.0 * width*width)).exp();
            let wave = (4.0*r - t*5.0).cos();
            gaussian * wave
        },
        [0.1, 0.8, 0.4], // base color
    );

// usage_example

Build a scene in a few lines

your_app.rs 
use wplot::{PlotData, PlotConfig, LegendEntry, plot_wireframe};

let range: Vec<f32> = (0..60)
    .map(|i| -5.0 + (i as f32 / 59.0) * 10.0)
    .collect();

let config = PlotConfig {
    grid_size: 12.0,
    grid_divisions: 12,
    legend: vec![
        LegendEntry { label: "sin(r)".into(),  color: [0.2, 0.5, 1.0] },
        LegendEntry { label: "ripple".into(), color: [1.0, 0.45, 0.15] },
    ],
    ..Default::default()
};

let plot_data = PlotData::new()
    .with_config(config)
    // Static surface
    .add_graph(plot_wireframe(
        &range, &range,
        |x, z| (x*x + z*z).sqrt().sin(),
        [0.2, 0.5, 1.0],
    ))
    // Animated surface
    .add_animated_graph(
        range.clone(), range.clone(),
        |x, z, t| {
            let r = (x*x + z*z).sqrt();
            (r - t * 2.5).sin() * (-r * 0.15).exp()
        },
        [1.0, 0.45, 0.15],
    );
// architecture

Clean, layered design

Each file has one job. The rendering pipeline never leaks into your data definitions.

FileResponsibility
main.rswinit event loop, input routing, scene definition
renderer.rswgpu device, pipelines, and render loop — the only file that touches the GPU
camera.rsSpherical-coordinate orbit camera with clamped pitch and zoom
geometry.rsMesh generation: 3-face grid box, wireframe surfaces, scatter clouds
mesh.rsCPU-side vertex/index storage and multi-mesh merging
data.rsPlotData builder and animated graph definitions
config.rsPlotConfig, legend entries, background color
vertex.rsGPU vertex layout — Float32x4 position + color, bytemuck-compatible
Drag Left-click + drag to orbit yaw & pitch
Scroll Mouse wheel to zoom in & out
Resize Viewport and depth buffer update automatically
